# AI 数据输出格式规范

> **适用场景：** AI_Helper 的用户级标准导入导出格式（非 `userData.json`）。
> **用途：** 当你（AI）被要求为 AI_Helper 生成提示词或补充提示词时，输出内容必须严格遵守以下格式。用户保存为 `.md` 或 `.txt` 后可通过菜单"文件 → 导入项目数据"导入。

---

## 格式规则

```
## 任务名称

**卡片名称**：内容

**卡片名称2**：内容第二行可以续写
多行内容直接续在冒号后即可。
```

**核心要素：**

| 要素 | 标记 | 说明 |
|------|------|------|
| 任务 | `## ` 二级标题 | 后跟任务名称，编号可选（如 `## 1. 赛博城市`）。每个任务独占一个 `## ` |
| 卡片 | `**卡片名**：` | 粗体包裹卡片名，紧跟中文或英文冒号，冒号后同行写内容。卡片名必须与标准卡片名称完全一致 |
| 续行 | 非标记行 | 不以 `##` 或 `**` 开头的行，自动追加到当前卡片的内容末尾（换行拼接） |

---

## 标准卡片名称（10 个固定维度）

必须使用以下名称（与软件 fieldConfig 一致）：

| 序号 | 卡片名称 | 说明 |
|------|---------|------|
| 1 | 主体特征 | 画面主体（人物、物体、生物等）的外貌、姿势、材质 |
| 2 | 场景环境 | 背景、地点、环境氛围 |
| 3 | 光影色彩 | 光源方向、色调、氛围光、对比度 |
| 4 | 艺术风格 | 画风、流派、渲染风格、胶片质感 |
| 5 | 镜头景别 | 构图、景别（特写/中景/远景）、角度 |
| 6 | 镜头运动 | 摄影机运动方式（推拉摇移跟升降） |
| 7 | 时间节奏 | 帧率、慢动作、延时等时间属性 |
| 8 | 动态事件 | 画面中的动作、事件、变化 |
| 9 | 技术参数 | 分辨率、画幅比、对焦等技术参数 |
| 10 | 负面排除 | 不希望出现的元素 |

> 超出 10 个固定维度的描述可使用**自定义卡片**，名称自定（如"声音设计"）。导入时软件自动识别匹配。

## 完整示例

### 单个任务

```markdown
## 1. 赛博朋克雨夜

**主体特征**：一位穿着黑色风衣的赛博格侦探，左眼是红色机械义眼，右臂露出银色金属骨骼，短发被雨水打湿。

**场景环境**：雨夜的霓虹街道，全息广告牌在林立的高楼间闪烁，地面倒映着紫色和蓝色的光，远处有悬浮车飞过。

**光影色彩**：冷色调为主，霓虹灯的红紫色光作为主光源，雨中反光营造迷离氛围。高对比度，阴影区域偏蓝。

**艺术风格**：赛博朋克风格，高对比度，电影级调色，Blade Runner 风格美学。

**镜头景别**：中景，半身构图，低角度仰拍，突出人物的孤独感和城市的压迫感。

**镜头运动**：从人物背后缓慢环绕至正面，手持微晃动增加纪实感。

**时间节奏**：慢动作，每秒 60 帧，雨滴下落速度减慢。

**动态事件**：人物点燃一支烟，烟雾在细雨中缓缓上升，霓虹灯光在烟雾中折射。

**技术参数**：4K，宽银幕 21:9，浅景深，锐利对焦，电影级色彩分级。

**负面排除**：无阳光，无自然植被，无噪点，无模糊，无文字，无水印。
```

### 多个任务

```markdown
## 1. 赛博朋克雨夜

**主体特征**：赛博格侦探，黑风衣，红色义眼。

**场景环境**：雨夜霓虹街道，全息广告牌。

**光影色彩**：冷色调，霓虹红紫光，高对比度。

## 2. 森林精灵

**主体特征**：精灵少女，绿色长发及腰，尖耳朵，穿着树叶和藤蔓编织的衣裳。

**场景环境**：清晨的魔法森林，阳光透过树冠形成束状光束，地面铺满苔藓和发光蘑菇。

**光影色彩**：暖金色为主，丁达尔光束穿过树叶，柔和的光斑洒在草地和精灵身上。

**艺术风格**：奇幻风格，吉卜力美学，柔和色系，手绘质感。

**镜头景别**：中全景，精灵置身于森林中，三分法构图。

**动态事件**：精灵轻轻触碰一朵发光蘑菇，蘑菇释放出飞舞的光粒子。
```

### TXT 格式（也支持导入）

```text
【1. 赛博城市】

主体特征：机械武士站立在霓虹灯下，全身液态装甲反射着城市的七彩光芒。

场景环境：赛博朋克风格的未来城市，高楼大厦林立，全息广告牌悬浮在空中。

光影色彩：青橙对比色，体积光穿过雨幕，逆光剪影效果。
```

## 严禁事项

1. **禁止** 使用 `# ` 一级标题
2. **禁止** 添加元信息行（如"导出时间"、"任务总数"等）
3. **禁止** 使用 `### ` 三级标题格式（那是卡片导入格式，与项目导入不兼容）
4. **禁止** 使用 `**内容**` 独占一行的标记方式（那是卡片导入格式）
5. **禁止** 在卡片名冒号前添加多余空格
6. **禁止** 使用 `---` 分隔线

---

## 导入方式

AI 生成的 `.md` 文件可通过以下方式导入：

- **文件菜单 → 导入项目数据** — 选择 .md 或 .txt 文件
- 支持同时解析 MD（`**卡片**：`）和 TXT（`卡片：`）混合格式
