## 格式 A：标准输入输出格式（通用导入格式）

**适用范围：** 工作台「导入数据」按钮、拖拽文件到任务项、右键菜单「导出任务」「导出卡片」

### 任务级别数据（含任务）

```markdown
## 任务名称

**卡片名称1**：这是卡片1的内容。
可以多行。

**卡片名称2**：这是卡片2的内容。
```

**规则：**
| 标记 | 含义 | 必填 |
|------|------|------|
| `## 任务名称` | 二级标题 = 一个任务的开始，后面是该任务的卡片 | 是（第一个 `##` 开始） |
| `**卡片名称**：内容` | 粗体 + 冒号 = 一张卡片的开始，同行冒号后为卡片数据 | 是 |

- 可以有多个 `##` 任务，每个任务下可以有多个 `**卡片名**：` 卡片
- 卡片数据从冒号后同行开始，可跨多行，直到下一个 `##` 或 `**卡片名**：`
- 卡片名称会与已有卡片按名称匹配。同名卡片可**跳过 / 覆盖 / 重命名**
- 任务名称同理，同名可**跳过 / 覆盖 / 重命名**

### 卡片级别数据（无任务）

如果文件中没有 `## ` 开头行，则解析为纯卡片数据，导入到**当前选中的任务**：

```markdown
**卡片名称1**：卡片1的内容。

**卡片名称2**：卡片2的内容。
```

## 格式 B：项目数据格式（MD）

**适用范围：** 菜单「文件 → 项目数据导出」、文件拖拽到侧边栏「新建任务 +」区域

```markdown
## 1. 任务A

**卡片名称1**：这是卡片1的内容。

**卡片名称2**：这是卡片2的内容。
可以多行。

## 2. 任务B

**卡片名称1**：任务B的卡片1内容。
```

**规则：**
| 标记 | 含义 |
|------|------|
| `## N. 任务名` | 带编号的二级标题，`N` 为从 1 开始的序号 |
| `**卡片名**：内容` | 粗体卡片名 + 冒号 + 同行内容。后续非标记行视为续行 |
| `---` / `======` | 分隔线，解析时忽略 |

**说明：**
- 冒号支持全角 `：` 和半角 `:`
- 任务编号 `N.` 可选，也支持不带编号的 `## 任务名`
- 固定卡片按 `fieldConfig` 顺序导出，自定义卡片按 `cardOrder` 顺序导出
- 隐藏卡片不导出

## 格式 C：项目数据格式（TXT）

**适用范围：** 菜单「文件 → 项目数据导出」（选择 .txt）

```text
【1. 任务A】
卡片名称1：这是卡片1的内容。
卡片名称2：这是卡片2的内容。
可以多行。

【2. 任务B】
卡片名称1：任务B的卡片1内容。
```

**规则：**
| 标记 | 含义 |
|------|------|
| `【N. 任务名】` | 全角方括号包裹，`N` 为从 1 开始的序号 |
| `卡片名：内容` | 纯文本卡片名 + 冒号 + 同行内容 |
| `---` / `======` | 分隔线，解析时忽略 |

**说明：**
- 以 `导出时间`、`导出任务`、`项目任务`、`任务总数` 等元信息开头的行被跳过
- 任务编号 `N.` 可选

## 格式对照表

| 场景 | 任务标记 | 卡片名称标记 | 内容标记 |
|------|---------|-------------|---------|
| **通用导入** (import.js) | `## 任务名` | `**卡片名**：` | 同行冒号后，可跨行 |
| **纯卡片导入** (import.js) | 无 | `**卡片名**：` | 同行冒号后，可跨行 |
| **项目数据 MD** (app.js) | `## 1. 任务名` | `**卡片名**：` | 同行冒号后 |
| **项目数据 TXT** (app.js) | `【1. 任务名】` | `卡片名：` | 同行冒号后 |
| **导出任务** (sidebar.js) | `## 1. 任务名` | `**卡片名**：` | 同行冒号后 |
| **导出卡片** (content.js) | 无 | `**卡片名**：` | 同行冒号后 |

> **注意：** 所有导入/导出路径现在统一使用 `**卡片名**：内容` 格式（粗体 + 冒号，同行立即可跟随卡片数据）。项目数据 TXT 格式使用 `卡片名：内容`（无粗体标记）。
